## 二、四大功能现状与目标

### 1. 搜索 (Search) — 现状 70%，目标 90%

**现在做到的：**
- 11个中文平台热榜采集（每30分钟）
- 9个中英文 RSS 源
- Mark当前内容已初步实现

**缺失的：**
- 英文社交平台（Twitter/X、Reddit、Hacker News）
- 深度文章提取（当前只抓标题，缺正文全貌）
- 意见领袖追踪（"插旗"——Musk、Buffett 等人的公开言论）
- 源质量评分（有的平台噪音高，有的信号强）

**12个月目标：**
```
多语言、多类型数据源，每个源有"信任评分"
每个意见领袖有"立场追踪"（bullish/bearish/neutral）
每个话题有时间线（最早出现时间、热度曲线、关键转折点）
```

### 2. 模拟 (Simulation) — 现状 20%，目标 60%

**现在做到的：**
- 基于动量和波动的回测引擎
- 参数优化器（网格搜索）

**缺失的：**
- 情景推演（"如果 X 发生，Y 会怎样？"）
- 多策略对抗（趋势跟踪 vs 均值回归 vs 事件驱动，谁在当前市场更有效？）
- 压力测试（极端行情下的组合表现）

**GitHub 可借鉴资源：**
- `microsoft/qlib` — AI 量化平台，含 Alpha 因子挖掘
- `vnpy/vnpy` — 量化交易框架
- `google/trax` — 强化学习用于交易策略

### 3. 预知 (Prediction) — 现状 15%，目标 80%

**现在做到的：**
- AI 分析生成 5 个板块的深度分析
- 决策引擎给出买入/卖出/持有建议

**缺失的 （最大差距）：**
- ❌ 预测注册表：每次预测都记录（何时、何因、何果），后续追踪准确率
- ❌ 历史模式库：当前信号与历史上哪些时期相似？当时发生了什么？
- ❌ 多Agent辩论：多个不同视角的AI互相挑战，产生更可靠的判断
- ❌ 预测集市：用户可以对预测投票、评论

**GitHub 可借鉴资源：**
- `langchain-ai/langgraph` — 多步骤推理链
- `joeddav/devol` — 遗传算法优化
- `EleutherAI/lm-evaluation-harness` — 评估框架

### 4. 收集 (Knowledge) — 现状 45%，目标 85%

**现在做到的：**
- Obsidian 笔记 + YAML frontmatter + wikilink
- ChromaDB 向量检索 + TF-IDF 回退
- 知识按月和分类组织

**缺失的：**
- ❌ 知识图谱：实体关系存储（公司A投资了公司B，政策C影响了行业D）
- ❌ 天才思想库：Musk/Buffett/Dalio 等人的公开言论、预测记录、事后验证
- ❌ 历史规律库：对过往市场周期、突发事件、政策变化的模式化总结
- ❌ 跨笔记推理：当前新闻与哪条历史笔记相关？关联到什么模式？

**GitHub 可借鉴资源：**
- `neo4j/neo4j` — 图数据库
- `mem0ai/mem0` — 长期记忆层
- `gchq/CoreNER` — 实体识别

## 三、"从未来倒退现在" — 倒推路线图

### 倒推逻辑

按照"12个月后要实现什么 → 6个月前必须完成什么 → 3个月前必须开始什么"的方式倒推。

```
2027-06: 预测准确率 > 80%，每天一条结构化预判消息
   ↑
2027-03: 知识图谱 + 多Agent辩论系统上线
  (需要: 实体关系库、多模型协作框架)
   ↑
2026-12: 预测注册表 + 历史模式库 + 英文源扩展
  (需要: 预测记录系统、模式匹配引擎、多语言爬虫)
   ↑
2026-09: 第2层LoRA群 + 哨兵全量部署 + 意见领袖追踪
  (需要: LoRA训练完成、哨兵稳定运行、数据源质量评分)
   ↑
2026-06: ✅ 哨兵上线 ✅ DeepSeek切换 ✅ AI日报简报
  (当前状态)
```

### 四线并进框架

每条线对应一个"百器"榫卯模块，但视角不同：

```
线A: 搜索 & 哨兵 (信息 → 信号)
  当前: 11中文平台 → 哨兵三模型标记
  Phase2: +Twitter/Reddit/HN → +相关性深度评分(LoRA-A)
  Phase3: +意见领袖插旗 → +源质量评分
  Phase4: +深度文章提取 → +多语言覆盖

线B: 预知 & 模拟 (信号 → 预判)  
  Phase1: ✅ 决策引擎8因子
  Phase2: 预测注册表(每次预测记下来)
  Phase3: 历史模式匹配(当前像历史上的哪个时期？)
  Phase4: 多Agent辩论(多模型多策略对抗)

线C: 收集 & 图谱 (预判 → 知识)
  Phase1: ✅ Obsidian + ChromaDB
  Phase2: 天才思想库(名人言论追踪)
  Phase3: 知识图谱(实体关系图)
  Phase4: 跨笔记推理自动关联

线D: 反馈 & 学习 (结果 → 归因)
  Phase1: ✅ Feedback_and_Learning 基础流程
  Phase2: 预判准确率仪表盘
  Phase3: 自动归因(为什么这次预测对了/错了？)
  Phase4: 策略自适应(归因结果自动调参)
```

### 四线协同示例

以一条新闻从采集到预判的全流程为例：

```
线A: 采集到"台积电CoWoS产能翻倍" → 哨兵标记[科技][positive][relevant]
                                              ↓
线B: 决策引擎评分 0.72 → 注册预测:"半导体设备股3个月内上涨概率65%"
                                              ↓
线C: 关联知识图谱 → 台积电→CoWoS→先进封装→受益公司列表
      关联历史模式 → 2023年CoWoS扩产时板块涨了40%
                                              ↓
线D: 3个月后验证 → 预测正确/错误 → 归因分析 → 调参
```

### 要追踪的"天才"（建议初始清单）

| 但斌 | A股投资 | 微博 | A股市场观点 |

### 思想与新闻的碰撞

当系统采集到新闻时，自动问：

1. **哪位"天才"对类似话题发表过看法？**
   → "Buffett 2024年说过类似的话：'现金是垃圾'"
   
2. **天才的过往预测准不准？**
   → "Musk 2023年预测FSD在18个月内实现，目前已过24个月没实现"

3. **当前新闻是印证还是反驳？**
   → "这个政策动向印证了Butt在股东会上说的'看好日本市场'"

## 五、"问题→珍贵资源"机制

### 核心理念

**"问题就是未被定价的机会"**

当系统发现一个"问题"时（矛盾、危机、争议），它标记的不是风险，而是**潜在资源**。

### 问题分类器

| 问题类型 | 示例 | 背后资源 |
|---------|------|---------|
| 监管争议 | 某行业面临监管不确定性 | 政策明朗化后的反弹机会 |
| 技术瓶颈 | 某个关键技术卡住了 | 突破后的爆发机会 |
| 市场恐慌 | 某板块被过度抛售 | 价值回归机会 |
| 认知分歧 | 分析师们观点严重分裂 | 真相浮出时的机会 |
| 信息不对称 | 小公司出了大事但没人报道 | 信息优势机会 |

### 实现方式

在哨兵中增加第4个模型 **S5-problem**: 判断新闻是否指向一个"问题"，以及问题类型。

## 七、关键决策原则

1. **先有追踪，后有预测** — 先建立预测注册表（记录），再追求准确率
2. **先有数据，后有模型** — 先积累天才言论数据，再训练观点分析模型
3. **先有框架，后有细节** — 先定义知识图谱的Schema，再填充实体
4. **一线优先** — 搜索与哨兵是其他三条线的基础，始终优先
5. **GitHub优先，自建其次** — 能用开源就不造轮子
