### 阶段 1：补充数据生成（服务器并行执行）

**目标：** 补足现有数据的短板（长文本、港股、宏观、多样性）

**前提条件：** `generate_v3.0.py` 已上传到服务器，API 密钥可用

**可并行执行的任务：**

| 任务 | 命令 | 目标量 | 预估时间 | 产出文件 | 并行？ |
|------|------|--------|----------|----------|--------|
| A. 混合模式 | `python3 generate_v3.0.py --mode mix --target 150000` | 15万条 | **8-12小时** | `generated_v3/train_v3_mix_*.jsonl` | ✅ |
| B. 港股补强 | `python3 generate_v3.0.py --mode hk-stock --target 30000` | 3万条 | **2-3小时** | `generated_v3/train_v3_hk-stock_*.jsonl` | ✅ (与A并行) |
| C. 下载模型 | `huggingface-cli download Qwen/Qwen2.5-7B-Instruct` | - | **10-30分钟** | `~/.cache/huggingface/` | ✅ (与A/B并行) |

> **并行策略：** A、B、C 三者无依赖关系，**可以同时在服务器上跑**。A和B各占最高 32 线程，但 MiMo API 限流会自然错开。C 是下载任务，几乎不占资源。

**执行步骤：**

```bash
# 1. 上传 v3.0 生成器（如果还没上传）
scp generate_v3.0.py root@8.140.232.52:/root/

# 2. 开第一个 SSH 窗口 → 跑混合模式（主要任务，时间长）
ssh root@8.140.232.52 "cd /root && nohup python3 generate_v3.0.py \
  --mode mix --target 150000 \
  > generate_mix.log 2>&1 &"

# 3. 开第二个 SSH 窗口 → 跑港股补强
ssh root@8.140.232.52 "cd /root && nohup python3 generate_v3.0.py \
  --mode hk-stock --target 30000 \
  > generate_hk.log 2>&1 &"

# 4. 开第三个 SSH 窗口 → 下载模型
ssh root@8.140.232.52 "huggingface-cli download Qwen/Qwen2.5-7B-Instruct"

# 5. 查进度
ssh root@8.140.232.52 "tail -f generate_mix.log"
```

### 阶段 2：数据处理与合并（服务器串行执行）

**目标：** 将所有数据源整合为 LLaMAFactory 训练格式

**前提条件：** 阶段 1 全部完成

| 输入 | 路径 | 操作 | 输出 |
|------|------|------|------|
| 旧数据(过滤后) | `/root/data_quality_report/train_data_filtered.jsonl` | 格式化 | `train_phase1.jsonl` |
| 新数据(混合模式) | `/root/generated_v3/train_v3_mix_*.jsonl` | 融合 | `train_phase1.jsonl` |
| 新数据(港股) | `/root/generated_v3/train_v3_hk-stock_*.jsonl` | 融合 | `train_phase1.jsonl` |
| 真实数据1.6万 | 本地项目中的路径 | 格式化+上采样 | `train_phase2.jsonl` |

**Phase 1 数据（继续预训练）：**

```yaml
# 数据组成
train_phase1.jsonl:
  - 过滤后旧数据:    1,245,107 条  (短文本，用于领域适应)
  - v3.0 混合模式:   150,000 条   (含长文本，丰富多样性)
  - v3.0 港股补强:    30,000 条   (补港股/宏观覆盖)
                         ─────────
  合计:              ~1,425,107 条
```

**Phase 2 数据（SFT 指令微调）：**

```yaml
# 数据组成
train_phase2.jsonl:
  - v3.0 长文本数据:  160,000 条  (从混合模式中提取type=long的)
  - 真实数据:          16,000 条  (5x上采样→80,000条等效)
                         ─────────
  合计:              ~240,000 条  (含上采样等效)
```

**执行命令：**

```bash
# 数据合并脚本（一次性命令）
python3 -c "
import json, glob, os

# ====== Phase 1 数据：所有数据用于继续预训练 ======
phase1 = []
# 旧数据（过滤后）
with open('/root/data_quality_report/train_data_filtered.jsonl') as f:
    for line in f:
        if line.strip():
            phase1.append(json.loads(line))

# v3.0 新数据
for pattern in ['/root/generated_v3/train_v3_mix_*.jsonl',
                '/root/generated_v3/train_v3_hk-stock_*.jsonl']:
    for fpath in glob.glob(pattern):
        with open(fpath) as f:
            for line in f:
                if line.strip():
                    phase1.append(json.loads(line))

print(f'Phase 1 数据: {len(phase1):,} 条')

# 输出
os.makedirs('/root/train_data', exist_ok=True)
with open('/root/train_data/train_phase1.jsonl', 'w') as f:
    for item in phase1:
        f.write(json.dumps(item, ensure_ascii=False) + '\n')

# ====== Phase 2 数据：仅用长文本 + 真实数据 ======
phase2 = []
for fpath in glob.glob('/root/generated_v3/train_v3_mix_*.jsonl'):
    with open(fpath) as f:
        for line in f:
            if line.strip():
                item = json.loads(line)
                # 只取长文本任务
                if item.get('meta', {}).get('type') == 'long':
                    phase2.append(item)

# 真实数据（假设在 /root/real_data.jsonl）
real_path = '/root/real_data.jsonl'
if os.path.exists(real_path):
    with open(real_path) as f:
        for line in f:
            if line.strip():
                phase2.append(json.loads(line))

print(f'Phase 2 数据: {len(phase2):,} 条')

with open('/root/train_data/train_phase2.jsonl', 'w') as f:
    for item in phase2:
        f.write(json.dumps(item, ensure_ascii=False) + '\n')
"
```
