  🔧 十、更多横切关注点

  1. 配置统一管理

  当前5个目录各有配置文件，环境变量分散。建议：
  - 统一配置中心（etcd/Consul 或简单点一个 shared_config.yaml）
  - 公共 .env.template 文件，CI 校验所有必需变量
  - 配置变更审计日志

  2. 文档体系

  - 每个服务独立的 README.md（启动方式、依赖、配置说明）
  - API 文档（OpenAPI/Swagger，特别对 invest-backend 和 semantic-search）
  - 架构决策记录（ADR）：记录关键架构决策和理由
  - 配置文件的注释说明（当前很多选项无注释，不知道是什么含义）

  3. 国际化（i18n）

  - 系统混合中英文（RSS 源中英文都有，AI 分析输出中文）
  - 增加统一的国际化方案：配置语言偏好，AI Prompt 根据目标语言调整

  4. 数据保留与隐私

  - 当前无数据清理策略。建议：
    - 原始采集数据：保留30天
    - 分析结果：保留90天
    - 决策历史：永久保留
    - 个人配置：用户可导出/删除
