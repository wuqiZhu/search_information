要让关键词帮你获取\*\*更真实、更高效\*\*的信息，不能只堆砌词条，而要从\*\*词的精准度、来源可信度、匹配策略、后续过滤\*\*等多个维度入手。下面我从 7 个角度给出具体建议。



\---



\## 一、关键词本身的优化



\### 1. 使用精确术语，减少泛化词

\- \*\*错误示范\*\*：只写“芯片”——会抓到大量无关内容（如手机芯片、家电芯片、芯片封装材料等）。

\- \*\*正确做法\*\*：根据你的关注点细化，如：`"AI芯片"、"自动驾驶芯片"、"算力芯片"`。

\- \*\*对于“华为”\*\*：可加上场景限定，如 `"华为 昇腾"、"华为 智能汽车"`，避免出现“华为手机销量”等无关信息。



\### 2. 利用正则或组合匹配

\- 比如 `"特斯拉\\s?(Optimus|机器人|FSD|Dojo)"` 可以更精准匹配特斯拉机器人或自动驾驶进展，而不是“特斯拉股价下跌”这类泛新闻。

\- 你的 `frequency\_words.txt` 支持正则，可以利用起来。



\### 3. 增加“必须词”和“排除词”组合

\- 例如：`+自动驾驶 +量产 -事故 -召回` → 确保内容是积极进展而非负面事件。

\- 但你当前 `frequency\_words.txt` 语法中，“+”表示必须词，“-”表示排除词（见你文档中语法说明）。可以灵活使用。



\---



\## 二、分层优先级管理



你可以把关键词分成三个等级，并在 `config.yaml` 中通过不同的来源或通知权重实现：



| 优先级 | 关键词示例 | 建议通知方式 |

|--------|-----------|--------------|

| \*\*高\*\*（行业突变） | `特斯拉 Optimus V3 发布`、`英伟达 新一代 GPU` | 实时钉钉推送 |

| \*\*中\*\*（常规进展） | `自动驾驶 量产`、`半导体 产能` | 每天汇总一次 |

| \*\*低\*\*（宏观背景） | `算力 需求`、`大模型 融资` | 存入数据库，周报查阅 |



你的 `TrendRadar` 已经支持按关键词设置不同的通知规则吗？如果没有，可以通过脚本先全部抓取，再由 `analyse\_information` 进行二次筛选。



\---



\## 三、来源筛选



真实性与来源强相关。宁可少而精，不要多而杂。



\### 建议优先使用的 RSS 源（你已配置部分）：

\- \*\*Phoronix\*\*、\*\*LWN.net\*\*、\*\*Hackaday\*\* → Linux/嵌入式/开源硬件新闻，可信度高。

\- \*\*TechCrunch AI\*\*、\*\*MIT Tech Review\*\* → AI 产业动态，相对严谨。

\- \*\*机器之心\*\*、\*\*量子位\*\*（中文）→ AI 领域深度报道，但注意标题党。

\- \*\*公司官方博客\*\*：如 OpenAI 博客、Tesla AI Day 页面。



\### 建议降级或禁用的源：

\- 今日头条、抖音热搜（噪声极大）

\- 微博热搜（娱乐化严重）

\- 百度热搜（广告多）



你可以在 `config.yaml` 中为不同源设置不同的 `enabled` 或 `layers`，让高优先级关键词只在高质量源中匹配。



\---



\## 四、利用 AI 二次过滤（你的 `analyse\_information` 已具备）



即使关键词匹配了，也可能抓来低质量文章。你的 `analyse\_information` 项目已经集成了 AI 相关性评估（0-10 分）和结构化分析。建议：



\- \*\*设置相关性阈值\*\*：比如只保留评分 ≥ 6 的文章，才进入知识库。

\- \*\*自动标记“疑似标题党”\*\*：在 AI prompt 中加入“检测文章是否属于标题党或夸张描述”，并给低分。

\- \*\*人工反馈闭环\*\*：你在钉钉通知里有点赞/忽略按钮，可以收集反馈，反向训练你的评分规则（虽然不能直接微调模型，但可以调整关键词权重或排除词）。



\---



\## 五、时效性与去重



\- \*\*时效性\*\*：对于“发布”、“首发”、“完成融资”等事件，关键词可以加上 `\\d{4}年\\d{1,2}月` 正则，避免抓到旧闻。但你的 RSS 通常会自动排序，所以优先级不必过高。

\- \*\*去重\*\*：你的 `TrendRadar` 已经支持数据库去重（按 URL 或标题哈希），确保同一条新闻不会重复推送。



\---



\## 六、多领域交叉验证



不要只依赖单一关键词。例如，“特斯拉 Optimus 机器人”可以从多个关键词交叉确认：

\- `特斯拉` + `机器人`

\- `Optimus` + `量产`

\- `人形机器人` + `特斯拉`



如果多个独立来源都在报道同一事件，可信度更高。你的系统可以设计一个简单的“热度”统计：相同主题（通过 TF-IDF 或相似度）在短期内出现多次，自动提升优先级。



\---



\## 七、定期回顾与微调



关键词不是一劳永逸的。建议：



\- \*\*每周花 15 分钟\*\*：浏览被推送的信号，把真正有价值的标记为“重要”，把误报的加入排除词或调整关键词。

\- \*\*每月一次\*\*：检查是否有新的重要技术名词或公司出现（比如“机器人”赛道新公司），及时补充。



\---



\## 💡 总结：你的关键词使用最佳实践



| 动作 | 具体做法 |

|------|----------|

| \*\*关键词精准\*\* | 使用 `AI芯片` 而非 `芯片`；`特斯拉 机器人` 而非 `特斯拉` |

| \*\*排除干扰\*\* | 增加 `!震惊`、`!重磅` 等排除词；利用 `-` 排除负面内容 |

| \*\*AI 二次评分\*\* | 设置相关性阈值 > 6 才入库；自动标记标题党 |

| \*\*交叉验证\*\* | 同一事件多关键词命中才推送 |

| \*\*定期优化\*\* | 每周复盘，调整关键词和排除词 |



你的系统已经很强，只需要在关键词精细度和来源质量上再下功夫，就能过滤掉 90% 的噪音，只留下真正对你有价值的信息。

## 💰 围绕4个核心需求的关键词策略



### 需求1：立马获得好的利益（学生身份→资源，省钱）

| 策略 | 具体做法 |
|------|----------|
| **品牌+学生** | 在 `frequency_words.txt` 中配置 `小米 学生优惠`、`华为 学生`、`大疆 学生优惠` |
| **平台+学生** | 配置 `支付宝 学生`、`美团 学生`、`京东 学生`（可自行添加） |
| **时间窗口** | 开学季（8-9月）、双11、618 是学生优惠高峰期，提前关注 |
| **竞赛时间** | 开源之夏（4-6月报名）、GSoC（2-4月报名）需要提前准备 |

**建议添加的关键词**（可在 `frequency_words.txt` 的品牌活动组中追加）：
```
支付宝 学生
美团 学生
京东 学生
Apple 教育优惠
微软 教育
```



### 需求2：减少信息差（抢占先机）

**核心思路**：第二层关键词已经覆盖了人事变动、融资上市、产品发布、政策福利。关键是确保**实时推送**配置正确。

| 策略 | 具体做法 |
|------|----------|
| **公司+事件** | 用MCP搜索特定公司的具体事件，如 `search_news(query="华为 裁员")` |
| **行业+事件** | 搜索行业级别的事件，如 `search_news(query="半导体 融资")` |
| **跨平台验证** | 同一事件在多个平台出现才确认为真，用 `aggregate_news()` |
| **时间敏感** | 人事变动、裁员等信息需要第一时间知道，确保第一二层通知不延迟 |



### 需求3：收集过去但还有用的信息

**核心思路**：第三层关键词抓取面经、源码、课程。这些信息不急，每日汇总即可。

| 策略 | 具体做法 |
|------|----------|
| **年份更新** | 每年更新面经关键词，如 `面经 2026` → `面经 2027` |
| **领域扩展** | 根据求职方向添加，如 `嵌入式 笔试`、`Linux 面试` |
| **历史搜索** | 用 MCP 的 `search_rss(keyword="面经", days=30)` 搜索过去30天的面经 |
| **知识库积累** | analyse_information 会自动将分析后的文章归档到 Obsidian 知识库 |



### 需求4：立刻获取信息的能力（公司深度搜索）

**核心思路**：通过 MCP Server 的现有工具组合，实现对公司信息的快速深度搜索。

**完整的公司调研流程**：

```
步骤1：搜索公司所有相关新闻
  → search_news(query="公司名", search_mode="entity", include_rss=True)

步骤2：查找关联事件
  → find_related_news(reference_title="公司名 关键事件", date_range="last_week")

步骤3：分析舆论趋势
  → analyze_topic_trend(topic="公司名", analysis_type="trend")

步骤4：情感分析（正面/负面）
  → analyze_sentiment(topic="公司名")

步骤5：跨平台聚合去重
  → aggregate_news(date_range="today")
```
