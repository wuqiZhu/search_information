## 一、项目总览

本项目由 4 个子项目组成，形成一个完整的信息采集→分析→应用的自动化系统：

| 项目 | 定位 | 技术栈 | 仓库 |
|------|------|--------|------|
| **search_information** | 信息采集 | Python + LiteLLM + SQLite | [GitHub](https://github.com/wuqiZhu/search_information) |
| **analyse_information** | 信息分析与知识沉淀 | Python + DeepSeek + Obsidian | 与 search_information 同仓库 |
| **find_job** | 求职自动化 | Python + Selenium + DeepSeek | [GitHub](https://github.com/wuqiZhu/find_job) |
| **invest** | 投资分析 | Python + Next.js + 东方财富API | [GitHub](https://github.com/wuqiZhu/invest) |

## 四、search_information（信息采集）

### 4.1 项目概述

TrendRadar 是一个全栈信息采集系统，支持 11 个热榜平台 + 20+ 个 RSS 源，配备 AI 深度分析和 10+ 通知渠道。

### 4.2 核心功能

| 功能 | 说明 |
|------|------|
| 热榜采集 | 今日头条、百度、微博、知乎、B站、抖音、澎湃、华尔街见闻等 11 个平台 |
| RSS 聚合 | Hacker News、掘金、牛客、Phoronix、LWN、Hackaday 等 20+ 源 |
| AI 分析 | 5 维深度分析：核心热点、舆论风向、异动信号、RSS 洞察、策略建议 |
| 关键词匹配 | 正则表达式、必须词、排除词、别名、限制条数 |
| 通知推送 | 钉钉、飞书、企业微信、Telegram、邮件、Bark、Slack、ntfy 等 |
| 数据存储 | SQLite（按日期分文件）+ HTML 报告 |
| MCP Server | 21 个 AI Agent 工具，支持自然语言查询 |

## 五、analyse_information（信息分析）

### 5.1 项目概述

信息分析与知识沉淀系统，从 RSS 源获取文章，通过 AI 进行相关性评估、结构化分析、自动分类，最终归档到 Obsidian 知识库。

### 5.2 核心功能

| 功能 | 说明 |
|------|------|
| RSS 拉取 | 从 TrendRadar 配置获取 RSS 源，去重后处理 |
| AI 统一分析 | 一次性完成相关性评估 + 结构化分析 + 分类 |
| 分步 Fallback | 统一分析失败时，分步执行（相关性→翻译→分析） |
| Obsidian 归档 | 自动生成 Markdown 笔记，按分类归档 |
| 每日/每周摘要 | 定期生成知识摘要 |
| 通知中心 | 优先级排序 + 去重 + 时间窗口合并 |

### 5.3 目录结构

```
analyse_information/
├── .env                              # 环境变量（DEEPSEEK_API_KEY）
├── analyzer/
│   ├── pipeline.py                   # 主流水线（RSS→分析→归档）
│   ├── ai_analyzer.py                # AI 分析（统一分析→分步fallback）
│   ├── rss_fetcher.py                # RSS 抓取
│   ├── knowledge_builder.py          # Obsidian 笔记构建
│   ├── digest.py                     # 每日/每周摘要
│   ├── db.py                         # SQLite 数据库
│   ├── config.yaml                   # 配置文件
│   ├── defuddle/                     # 内容提取（Defuddle + BS4 fallback）
│   └── quality/                      # 内容质量验证
├── shared/
│   ├── notification_center.py        # 通知中心
│   └── notification_config.yaml      # 通知配置
├── knowledge_base/
│   ├── obsidian/                     # Obsidian 知识库
│   ├── raw/                          # 原始数据
│   └── analyzed/                     # 分析结果
├── tests/                            # 单元测试
└── Dockerfile                        # Docker 部署
```

### 5.4 AI 分析流程

```
文章 URL
    │
    v
Defuddle 内容提取（失败降级到 BeautifulSoup）
    │
    v
AI 统一分析（1次 API 调用）
    ├─ 相关性评估（0-10）
    ├─ 结构化分析（要点、观点、数据、启发）
    └─ 自动分类（政策福利/行业动态/嵌入式/Linux/...）
    │
    ├─ 成功 → 生成 Obsidian 笔记
    │
    └─ 失败 → 分步 Fallback
         ├─ 相关性判断（1次调用）
         ├─ 翻译（1次调用）
         └─ 结构化分析（1次调用）
```

### 5.5 配置文件

| 文件 | 用途 |
|------|------|
| `analyzer/config.yaml` | RSS源、AI模型、相关性规则、分类规则、Obsidian路径 |
| `shared/notification_config.yaml` | 钉钉 Webhook、优先级规则 |
| `.env` | DeepSeek API Key |

### 5.6 启动命令

```powershell
# 本地运行
cd analyse_information
python analyzer/pipeline.py --rss

# Docker 运行（服务器）
docker-compose up -d analyser
```

## 六、find_job（求职自动化）

### 6.1 项目概述

多平台求职岗位抓取系统，支持 6 个招聘平台，通过 AI 进行岗位匹配评分，高分岗位通过钉钉 ActionCard 推送（带投递按钮）。

### 6.2 核心功能

| 功能 | 说明 |
|------|------|
| 多平台抓取 | Boss直聘、猎聘、拉勾、牛客、智联、实习僧 |
| AI 评分 | 基于简历的岗位匹配评分（0-100分） |
| 钉钉通知 | 高分岗位 ActionCard（带投递按钮）、普通岗位 Markdown |
| 投递反馈 | 记录投递状态（已投递/不合适/已查看） |
| 去重机制 | seen_jobs.json 避免重复推送 |
| 反馈学习 | 根据投递结果调整评分偏好 |

### 6.3 搜索查询

46 个搜索查询，覆盖多城市、多岗位类型：

```python
"嵌入式软件工程师 应届"
# ... 共 46 个
```

### 6.5 AI 评分流程

```
岗位 JD 文本
    │
    v
构建 Prompt（简历 + JD + 评分标准）
    │
    v
DeepSeek API 调用（限制：30次/运行，间隔5-10秒）
    │
    v
解析 JSON 响应
    ├─ score: 0-100
    ├─ match_reasons: 匹配原因
    ├─ missing_skills: 缺失技能
    └─ advice: 建议
    │
    v
评分 ≥ 80 → 钉钉 ActionCard（带投递按钮）
评分 < 80 → 钉钉 Markdown（简要信息）
```
