## 二、启动顺序

### 第一步：启动 Docker 服务

```powershell
# 启动 n8n（自动化工作流）
cd search_information\n8n
docker-compose up -d

# 验证 n8n 运行
docker ps | findstr n8n
# 访问 http://localhost:5678（账号 admin / n8n123）
```

### 第二步：启动 TrendRadar（信号采集）

```powershell
cd search_information\TrendRadar
python main.py
```

TrendRadar 启动后会：
- 每 30 分钟自动抓取一轮信号
- 匹配关键词的信号会保存到 `data/signals/`
- P0/P1 级信号立即推送到钉钉
- P2 级信号等到每日报告时发送
- 信号同时通过 Webhook 推送到 n8n

### 第三步：启动 analyse_information（数据处理）

```powershell
# RSS拉取和AI分析
cd analyse_information
python analyzer/pipeline.py --rss

# 生成每日摘要
python analyzer/pipeline.py --digest
```

## 三、Windows 任务计划程序设置

### 3.1 打开任务计划程序

1. 按 `Win + S`，搜索 `任务计划程序`
2. 或按 `Win + R`，输入 `taskschd.msc`，回车

### 3.2 创建任务：TrendRadar 信号采集（开机自启）

**用途**：电脑开机后自动启动 TrendRadar，持续监控信号源

**步骤**：

1. 右侧点击 `创建基本任务`
2. 名称：`TrendRadar 信号采集`
3. 描述：`开机自动启动 TrendRadar 监控嵌入式Linux技术信号`
4. 触发器：`当计算机启动时`
5. 操作：`启动程序`
6. 程序或脚本：
   ```
   search_information\TrendRadar\start_trendradar.bat
   ```
7. 完成

**注意**：`start_trendradar.bat` 内容如下（使用相对路径）：

```bat
@echo off
chcp 65001 >nul
cd /d "%~dp0"
python main.py
```

### 3.3 创建任务：n8n Docker 启动（开机自启）

**用途**：电脑开机后自动启动 n8n Docker 容器

**步骤**：

1. 创建基本任务
2. 名称：`n8n Docker 启动`
3. 触发器：`当计算机启动时`
4. 操作：`启动程序`
5. 程序或脚本：
   ```
   search_information\n8n\start_n8n.bat
   ```
6. 完成

**`n8n/start_n8n.bat` 内容**：

```bat
@echo off
chcp 65001 >nul
cd /d "%~dp0"
docker-compose up -d
```

### 3.4 创建任务：analyse_information RSS处理（每小时）

**用途**：每小时自动拉取 RSS 并进行 AI 分析

**步骤**：

1. 创建基本任务
2. 名称：`analyse_information RSS处理`
3. 触发器：`每天`，然后在高级设置中重复任务间隔设为 `1 小时`，持续时间设为 `无限期`
4. 操作：`启动程序`
5. 程序或脚本：
   ```
   analyse_information\start_analyser.bat
   ```
6. 完成

### 3.5 创建任务：analyse_information 每日摘要（每天 23:00）

**用途**：每天晚上生成当日摘要

**步骤**：

1. 创建基本任务
2. 名称：`analyse_information 每日摘要`
3. 触发器：`每天`，时间设为 `23:00`
4. 操作：`启动程序`
5. 程序或脚本：
   ```
   analyse_information\start_digest.bat
   ```
6. 完成

### 3.6 任务计划程序设置汇总

| 任务名称 | 触发器 | 程序/脚本 | 说明 |
|----------|--------|-----------|------|
| TrendRadar 信号采集 | 计算机启动时 | `search_information\TrendRadar\start_trendradar.bat` | 开机自启，持续运行 |
| n8n Docker 启动 | 计算机启动时 | `search_information\n8n\start_n8n.bat` | 开机自启 Docker 容器 |
| analyse_information RSS处理 | 每天，每1小时重复 | `analyse_information\start_analyser.bat` | 每小时拉取RSS并分析 |
| analyse_information 每日摘要 | 每天 23:00 | `analyse_information\start_digest.bat` | 每晚生成摘要 |

## 四、日常使用流程

### 4.1 开机后（自动）

任务计划程序会自动启动：
- n8n Docker 容器
- TrendRadar 信号监控

### 4.2 查看信号（钉钉）

TrendRadar 每 30 分钟抓取一轮，匹配关键词的信号会自动推送到钉钉群：
- P0 级：立即单独推送
- P1 级：10 分钟内合并推送
- P2 级：包含在每日报告中

### 4.3 查看知识库（Obsidian）

打开 Obsidian，查看 `knowledge_base/obsidian/` 目录：
- AI 分析后的文章会按分类自动归档
- 10 个分类目录对应金字塔模型的各个层级

### 4.4 查看日志

```powershell
# TrendRadar 日志
Get-Content search_information\TrendRadar\logs\trendradar.log -Tail 50

# analyse_information 日志
Get-Content analyse_information\shared\logs\scheduler.log -Tail 50
```

## 五、手动操作场景

### 场景1：修改关键词后重启 TrendRadar

```powershell
# 1. 编辑配置文件
notepad search_information\TrendRadar\config.yaml

# 2. 停止旧进程
taskkill /F /IM python.exe

# 3. 清理已处理信号（可选，让已处理的信号重新匹配）
Remove-Item search_information\TrendRadar\data\processed_signals.json -Force

# 4. 重新启动
cd search_information\TrendRadar
python main.py
```

### 场景2：手动触发 analyse_information 分析

```powershell
cd analyse_information

# 分析单个URL
python analyzer/pipeline.py --url "https://www.kernel.org/doc/html/latest/driver-api/gpio/index.html"

# 拉取RSS并分析
python analyzer/pipeline.py --rss

# 生成每日摘要
python analyzer/pipeline.py --digest

# 查看统计信息
python analyzer/pipeline.py --stats
```

### 场景3：测试钉钉通知

```powershell
cd search_information\TrendRadar
python -c "
import hmac, hashlib, base64, urllib.parse, time, requests, os
from dotenv import load_dotenv
load_dotenv('../.env')
webhook = os.environ['DINGTALK_WEBHOOK']
secret = os.environ['DINGTALK_SECRET']
ts = str(round(time.time() * 1000))
ss = f'{ts}\n{secret}'
sign = urllib.parse.quote_plus(base64.b64encode(hmac.new(secret.encode(), ss.encode(), hashlib.sha256).digest()))
payload = {'msgtype':'markdown','markdown':{'title':'TrendRadar 测试通知','text':'## 测试通知\n\n如果你看到这条消息，说明配置正确！'}}
r = requests.post(f'{webhook}&timestamp={ts}&sign={sign}', json=payload, timeout=10)
print(r.text)
"
```

### 场景4：重启 n8n

```powershell
cd search_information\n8n
docker-compose down
docker-compose up -d
```

### 场景5：查看 Docker 容器状态

## 六、配置文件修改指南

### 6.1 修改 TrendRadar 关键词

编辑 `search_information/TrendRadar/config.yaml` 中的 `keywords` 部分：

```yaml
keywords:
  job_hunting:    # 第1层：求职招聘（权重1.5）
    - "实习"
    - "校招"
    # 添加更多...
  industry:       # 第4层：行业动态（权重0.8）
    - "嵌入式"
    - "Linux内核"
    # 添加更多...
```

### 6.2 修改 TrendRadar 信号源

编辑 `search_information/TrendRadar/config.yaml` 中的 `sources` 部分：

```yaml
sources:
  rss:
    - name: "新源名称"
      url: "https://example.com/feed"
      enabled: true
  web:
    - name: "新网页源"
      url: "https://example.com"
      selector: ".article-list"
      enabled: true
```

### 6.3 修改钉钉通知关键词

如果修改了钉钉机器人的安全关键词，需要同步修改 `main.py` 中所有消息标题：

```python
# 搜索 "通知" 关键词，替换为新的关键词
# 文件位置：TrendRadar/main.py
```

### 6.4 修改 analyse_information 配置

编辑 `analyse_information/analyzer/config.yaml`：
- `rss.feeds`：RSS 源列表
- `relevance.keywords`：关键词配置
- `categories`：分类规则

## 七、故障排查

### 问题1：TrendRadar 获取 0 个信号

**可能原因**：
1. 关键词太长太精确，无法匹配
2. 相关性阈值太高
3. 网络问题导致 RSS 源无法访问

**解决方案**：
```powershell
# 1. 降低相关性阈值
# 编辑 config.yaml，修改 min_relevance_score 为更小的值（如 0.15）

# 2. 使用短关键词
# 将 "嵌入式Linux实习" 改为 "嵌入式"、"实习"

# 3. 检查网络
python -c "import feedparser; f=feedparser.parse('https://www.phoronix.com/rss.php'); print(len(f.entries))"
```

### 问题2：钉钉收不到通知

**可能原因**：
1. 钉钉机器人安全模式不匹配
2. Webhook URL 错误
3. 环境变量未加载

**解决方案**：
```powershell
# 1. 测试钉钉连接
# 使用上面的"测试钉钉通知"脚本

# 2. 检查 .env 文件
cat search_information\.env | findstr DINGTALK

# 3. 检查钉钉机器人设置
# 确认安全模式（关键词/加签/IP白名单）
# 如果是关键词模式，确保消息中包含该关键词
```

### 问题3：n8n 无法访问

**可能原因**：
1. Docker 容器未启动
2. 端口未映射

**解决方案**：
```powershell
# 1. 检查容器状态
docker ps | findstr n8n

# 2. 如果容器不存在，重新启动
cd search_information\n8n
docker-compose up -d

# 3. 检查端口映射
docker port n8n
# 应该显示 5678/tcp -> 0.0.0.0:5678
```

### 问题4：analyse_information 分析失败

**可能原因**：
1. API Key 过期或错误
2. 网络无法访问 AI API

**解决方案**：
```powershell
# 1. 检查 API Key
cat analyse_information\.env | findstr API

# 2. 测试 API 连接
python -c "import requests; r=requests.post('https://api.deepseek.com/v1/chat/completions', headers={'Authorization': 'Bearer YOUR_KEY'}, json={'model':'deepseek-chat','messages':[{'role':'user','content':'test'}]}); print(r.status_code)"
```

## 八、数据备份

### 备份内容

| 文件/目录 | 说明 | 备份频率建议 |
|-----------|------|-------------|
| `.env` | 所有密钥和配置 | 修改后立即备份 |
| `TrendRadar/config.yaml` | TrendRadar 配置 | 修改后备份 |
| `analyse_information/analyzer/config.yaml` | 分析器配置 | 修改后备份 |
| `TrendRadar/data/signals/` | 历史信号数据 | 每周备份 |
| `analyse_information/knowledge_base/` | 知识库 | 每周备份 |
| `TrendRadar/logs/` | 运行日志 | 按需备份 |

### 备份命令

```powershell
# 备份配置文件
Copy-Item search_information\.env C:\backup\search_information_env.bak
Copy-Item search_information\TrendRadar\config.yaml C:\backup\trendradar_config.bak

# 备份信号数据
Copy-Item -Recurse search_information\TrendRadar\data C:\backup\trendradar_data

# 备份知识库
Copy-Item -Recurse analyse_information\knowledge_base C:\backup\knowledge_base
```
