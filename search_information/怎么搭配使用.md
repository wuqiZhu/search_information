# 四个项目搭配使用指南

最后更新：2026-05-25

---

## 一、项目总览

| 项目 | 定位 | 核心功能 |
|------|------|----------|
| **search_information** (TrendRadar) | 信息采集 | 11个热榜平台 + RSS聚合 + AI深度分析 + 10+通知渠道 |
| **analyse_information** | 信息分析与知识沉淀 | URL内容提取 + AI相关性评估 + Obsidian知识库归档 |
| **find_job** | 求职自动化 | 6个招聘平台抓取 + AI岗位评分 + 钉钉通知 + 投递反馈追踪 |
| **invest** | 投资分析 | 基金数据获取 + 技术分析 + 交易策略 + 钉钉通知 |

---

## 二、数据流总览

```
search_information（信息采集）        analyse_information（信息分析）        find_job（求职自动化）        invest（投资分析）
┌─────────────────────────┐        ┌─────────────────────────┐        ┌─────────────────────────┐        ┌─────────────────────────┐
│  TrendRadar             │        │  analyzer/pipeline.py   │        │  job_scraper/main.py    │        │  main_cli.py            │
│  ├─ 热榜平台 (11个)     │        │  ├─ RSS拉取 + AI分析    │        │  ├─ Boss直聘 (浏览器)   │        │  ├─ 基金数据获取        │
│  ├─ RSS源 (12个)        │  ──>   │  ├─ 每日/每周摘要       │  ──>   │  ├─ 猎聘/拉勾/牛客      │        │  ├─ 技术分析            │
│  └─ AI深度分析          │  RSS   │  └─ Obsidian归档        │  共享  │  ├─ 智联/实习僧         │        │  ├─ 交易策略            │
│                         │        │                         │  通知  │  └─ AI评分 + 反馈追踪   │        │  └─ 钉钉通知            │
│  钉钉/飞书/企微 ← 通知  │        │  钉钉通知 ← 分析结果    │  中心  │  钉钉通知 ← 高分岗位    │        │  钉钉通知 ← 投资信号    │
└─────────────────────────┘        └─────────────────────────┘        └─────────────────────────┘        └─────────────────────────┘
         │                                      │                               │                               │
         v                                      v                               v                               v
   SQLite + S3存储                        Obsidian知识库                   data/seen_jobs.json              fund_data.db
   output/news/*.db                  knowledge_base/obsidian/          data/feedback_records.json        基金历史数据+分析结果
```

四项目通过以下方式联动：
- **RSS源共享**：analyse_information 的 rss_fetcher 引用 TrendRadar 的 config.yaml 获取 RSS 源
- **通知中心共享**：find_job 的 notification_client 接入 analyse_information 的通知中心（优先级排序 + 去重 + 时间窗口合并）
- **信号目录共享**：analyse_information 可处理 TrendRadar 的 data/signals/ 目录
- **钉钉通知独立**：invest 使用独立的钉钉通知，与 analyse_information 的通知中心分离

## 三、启动顺序

### 第一步：启动 Docker 服务（可选）

```powershell
# 启动 n8n（自动化工作流，如不需要可跳过）
cd search_information\n8n
docker-compose up -d

# 验证 n8n 运行
docker ps | findstr n8n
# 访问 http://localhost:5678（账号 admin / n8n123）
```

### 第二步：启动 TrendRadar（信息采集）

```powershell
cd search_information\TrendRadar
python main.py
```

TrendRadar 启动后会：
- 抓取 11 个热榜平台 + 12 个 RSS 源的数据
- AI 深度分析热点趋势（5维分析：核心热点/舆论风向/异动信号/RSS洞察/策略建议）
- 匹配关注词的新闻推送到钉钉/飞书/企微等渠道
- 数据存储到 SQLite（按日期分文件）

### 第三步：启动 analyse_information（信息分析）

```powershell
# RSS拉取和AI分析
cd analyse_information
python analyzer/pipeline.py --rss

# 分析单个URL
python analyzer/pipeline.py --url "https://example.com/article"

# 生成每日摘要
python analyzer/pipeline.py --digest

# 生成每周摘要
python analyzer/pipeline.py --weekly-digest
```

### 第四步：启动 find_job（求职抓取）

```powershell
# 方式1：使用新模块化入口（推荐）
cd find_job
python -m job_scraper.main

# 方式2：使用启动脚本
find_job\start_find_job.bat
```

find_job 启动后会：
- 依次搜索 47 个查询（Boss直聘 + 猎聘 + 拉勾 + 牛客 + 智联 + 实习僧）
- 对新岗位进行 AI 匹配评分（基于 profile.json 中的简历）
- 高分岗位（≥80分）通过钉钉 ActionCard 推送（带投递按钮）
- 自动过滤黑名单岗位
- 记录投递反馈（已投递/不合适/已查看）

### 第五步：启动 invest（投资分析）

```powershell
# 分析所有配置的基金
cd invest\scripts
python main_cli.py analyze

# 监控基金（带通知）
python main_cli.py monitor --notify

# 查看基金持仓
python main_cli.py portfolio

# 每日分析（推荐定时执行）
python main_cli.py daily
```

invest 启动后会：
- 从东方财富等平台获取基金净值数据
- 进行技术分析（MA、RSI、布林带、MACD等）
- 根据交易策略生成买卖信号
- 通过钉钉推送投资信号和分析报告

## 四、Windows 任务计划程序设置

### 4.1 打开任务计划程序

1. 按 `Win + S`，搜索 `任务计划程序`
2. 或按 `Win + R`，输入 `taskschd.msc`，回车

### 4.2 创建任务：TrendRadar 信号采集（开机自启）

1. 右侧点击 `创建基本任务`
2. 名称：`TrendRadar 信号采集`
3. 触发器：`当计算机启动时`
4. 操作：`启动程序`
5. 程序或脚本：`search_information\TrendRadar\start_trendradar.bat`

### 4.3 创建任务：n8n Docker 启动（开机自启，可选）

1. 创建基本任务
2. 名称：`n8n Docker 启动`
3. 触发器：`当计算机启动时`
4. 操作：`启动程序`
5. 程序或脚本：`search_information\n8n\start_n8n.bat`

### 4.4 创建任务：analyse_information RSS处理（每小时）

1. 创建基本任务
2. 名称：`analyse_information RSS处理`
3. 触发器：`每天`，高级设置中重复任务间隔设为 `1 小时`，持续时间设为 `无限期`
4. 操作：`启动程序`
5. 程序或脚本：`analyse_information\start_analyser.bat`

### 4.5 创建任务：analyse_information 每日摘要（每天 23:00）

1. 创建基本任务
2. 名称：`analyse_information 每日摘要`
3. 触发器：`每天`，时间设为 `23:00`
4. 操作：`启动程序`
5. 程序或脚本：`analyse_information\start_digest.bat`

### 4.6 创建任务：find_job 岗位抓取（每天 8:00 和 20:00）

1. 创建基本任务
2. 名称：`find_job 岗位抓取`
3. 触发器：`每天`，时间设为 `8:00`，然后在高级设置中重复任务间隔设为 `12 小时`，持续时间设为 `无限期`
4. 操作：`启动程序`
5. 程序或脚本：`find_job\start_find_job.bat`

**也可以通过 GitHub Actions 自动执行**（推荐）：
- 在 GitHub 仓库的 Actions 页面手动触发 `Job Scraper`
- 或等待 cron 调度（默认每天 UTC 0:01，即北京时间 8:01）

### 4.7 创建任务：invest 每日分析（每天 18:00）

1. 创建基本任务
2. 名称：`invest 每日分析`
3. 触发器：`每天`，时间设为 `18:00`（A股收盘后）
4. 操作：`启动程序`
5. 程序或脚本：`invest\scripts\daily_analysis.bat`

### 4.8 任务计划程序设置汇总

| 任务名称 | 触发器 | 程序/脚本 | 说明 |
|----------|--------|-----------|------|
| TrendRadar 信号采集 | 计算机启动时 | `search_information\TrendRadar\start_trendradar.bat` | 开机自启，持续运行 |
| n8n Docker 启动 | 计算机启动时 | `search_information\n8n\start_n8n.bat` | 开机自启 Docker 容器（可选） |
| analyse_information RSS处理 | 每天，每1小时重复 | `analyse_information\start_analyser.bat` | 每小时拉取RSS并分析 |
| analyse_information 每日摘要 | 每天 23:00 | `analyse_information\start_digest.bat` | 每晚生成摘要 |
| find_job 岗位抓取 | 每天 8:00 和 20:00 | `find_job\start_find_job.bat` | 每天两次岗位抓取和评分 |
| invest 每日分析 | 每天 18:00 | `invest\scripts\daily_analysis.bat` | A股收盘后分析基金 |

## 五、日常使用流程

### 5.1 开机后（自动）

任务计划程序会自动启动：
- n8n Docker 容器（如果配置了）
- TrendRadar 信号监控（持续运行）

### 5.2 查看热点新闻（钉钉/飞书/企微）

TrendRadar 会持续抓取并推送：
- 匹配关注词的热点新闻实时推送
- AI 深度分析报告定期推送
- 支持 10+ 通知渠道

### 5.3 查看知识库（Obsidian）

打开 Obsidian，查看 `knowledge_base/obsidian/` 目录：
- AI 分析后的文章会按分类自动归档
- 支持相关笔记推荐
- 支持反馈学习（标记有用/无用后，AI 会调整后续分析偏好）

### 5.4 查看求职信息（钉钉）

每天 8:00 和 20:00 自动抓取岗位：
- 高分岗位（≥80分）：ActionCard 通知，带"立即投递"按钮
- 普通岗位：Markdown 通知，简要信息
- 数据文件：`find_job/data/seen_jobs.json`（已见岗位）、`find_job/data/feedback_records.json`（反馈记录）

### 5.5 查看投资分析（钉钉）

每天 18:00 自动分析基金：
- 技术指标分析（MA、RSI、布林带、MACD）
- 交易信号推送（买入/卖出/持有）
- 风险预警（最大回撤、波动率异常）
- 数据库：`invest/scripts/fund_data.db`

### 5.6 查看日志

```powershell
# TrendRadar 日志
Get-Content search_information\TrendRadar\logs\trendradar.log -Tail 50

# analyse_information 日志
Get-Content analyse_information\shared\logs\scheduler.log -Tail 50

# find_job 日志
Get-Content find_job\data\logs\scraper.log -Tail 50

# invest 日志
Get-Content invest\scripts\logs\fund_analysis.log -Tail 50
```

## 六、手动操作场景

### 场景1：修改 TrendRadar 关注词

```powershell
# 编辑关注词配置
notepad search_information\TrendRadar\config\frequency_words.txt

# 关注词语法：
# [WORD_GROUPS]          - 词组定义区域
# 华为                   - 简单关键词
# /正则表达式/           - 正则匹配
# +必须词                - 必须包含
# !排除词                - 排除包含
# => 别名               - 显示别名
# [组名]                 - 词组标签
# @5                     - 限制条数

# 重启 TrendRadar 使配置生效
```

### 场景2：修改 TrendRadar 热榜平台

编辑 `search_information/TrendRadar/config/config.yaml` 中的 `platforms` 部分：

```yaml
platforms:
  weibo:
    enabled: true
    interval: 300
  zhihu:
    enabled: true
    interval: 300
  # 禁用某个平台：设置 enabled: false
```

### 场景3：手动触发 analyse_information 分析

```powershell
cd analyse_information
```

## 七、配置文件修改指南

### 7.1 search_information 配置

| 配置文件 | 用途 |
|----------|------|
| `config/config.yaml` | 主配置：热榜平台、RSS源、通知渠道、AI分析、存储 |
| `config/frequency_words.txt` | 关注词配置（支持正则/必须词/排除词） |
| `config/ai_analysis_prompt.txt` | AI 分析提示词 |
| `.env` | 敏感配置（API Key、Webhook URL 等） |

### 7.2 analyse_information 配置

| 配置文件 | 用途 |
|----------|------|
| `analyzer/config.yaml` | 主配置：RSS源、AI模型、相关性判断、分类规则、Obsidian路径 |
| `shared/notification_config.yaml` | 通知中心配置：钉钉 Webhook、优先级规则、合并窗口 |
| `.env` | 敏感配置（DeepSeek API Key 等） |

### 7.3 find_job 配置

| 配置文件 | 用途 |
|----------|------|
| `profile.json` | 个人简历配置：教育背景、技能、项目、评分标准、黑名单 |
| `config.yaml` | 平台配置（目前主要通过环境变量控制） |
| `.env` | 敏感配置（BOSS_COOKIE、DEEPSEEK_API_KEY、DINGTALK_WEBHOOK 等） |
| `job_scraper/constants.py` | 搜索查询列表（47个查询）和阈值常量 |

### 7.4 invest 配置

| 配置文件 | 用途 |
|----------|------|
| `scripts/config.yaml` | 主配置：基金列表、分析参数、交易策略、通知设置 |
| `scripts/user_config.yaml` | 用户自定义配置（覆盖默认配置） |
| `.env` | 敏感配置（API Key、Webhook URL 等） |

### 7.5 钉钉机器人安全设置

四个项目的钉钉通知需要确保消息中包含机器人设置的安全关键词。默认关键词为"通知"。如果修改了钉钉机器人的安全关键词，需要同步修改各项目的提示词/模板。

## 八、环境变量配置

```env
# Boss直聘 Cookie（必须，否则无法抓取）

# 抓取模式：auto（先浏览器后HTTP）、browser（仅浏览器）、http（仅HTTP）

ZHAOPIN_COOKIE=xxx
ZHAOPIN_AT=xxx
ZHAOPIN_RT=xxx
```

## 九、故障排查

### 问题1：TrendRadar 获取 0 个新闻

**可能原因**：
1. 关注词太长太精确，无法匹配
2. 热榜平台接口变化
3. 网络问题

**解决方案**：
```powershell
# 1. 降低匹配要求
# 编辑 config/frequency_words.txt，使用更短的关键词

# 2. 检查网络连接
python -c "import requests; r=requests.get('https://newsnow.busiyi.world/api/s?platform=weibo'); print(r.status_code, len(r.json().get('data',[])))"
```

### 问题2：钉钉收不到通知

**可能原因**：
1. 钉钉机器人安全模式不匹配（关键词/加签/IP白名单）
2. Webhook URL 错误
3. 环境变量未加载

**解决方案**：
```powershell
# 1. 检查 .env 文件
cat .env | findstr DINGTALK

# 2. 测试钉钉连接（find_job）
cd find_job
python -c "from job_scraper.notifier import send_markdown; print(send_markdown('测试', '## 测试通知'))"

# 3. 检查钉钉机器人设置
# 确认安全模式，确保消息中包含安全关键词
```

### 问题3：find_job 抓取失败

**可能原因**：
1. BOSS_COOKIE 过期或无效
2. 触发了安全验证
3. DeepSeek API Key 无效

**解决方案**：
```powershell
# 1. 更新 Cookie
cd find_job
python scripts/get_cookie.py

# 2. 检查 Cookie 状态
cat .env | findstr BOSS_COOKIE

# 3. 检查 API Key
cat .env | findstr DEEPSEEK

# 4. 查看详细日志
Get-Content data\logs\scraper.log -Tail 30
```

### 问题4：find_job Cookie 异常通知

如果收到"Cookie异常提醒"通知：
