# 金字塔关键词模型

最后更新：2026-05-26

关键词配置文件：`search_information/TrendRadar/config/frequency_words.txt`

---

## 设计理念

整个关键词系统围绕 **4个核心需求** 设计：

| 需求 | 定位 | 对应层级 |
|------|------|----------|
| 1. 立马获得好的利益 | 学生身份→资源，小米活动省钱 | 第一层 |
| 2. 减少信息差 | 一手/二手信息，抢占先机 | 第二层 |
| 3. 收集过去但还有用的信息 | 面经、源码、课程 | 第三层 |
| 4. 立刻获取信息的能力 | 公司实际情况、部门关系 | 第四层（MCP） |
| 背景参考 | 产业趋势，辅助决策 | 第五层 |

---

## 金字塔模型详解

```
                    ▲
                   /  \
                  / 第1层 \        ← 立即获利（实时推送）
                 / 省钱资源 \
                /────────────\
               /    第2层     \    ← 抢占先机（实时推送）
              /   减少信息差   \
             /────────────────\
            /      第3层       \   ← 经验资源（每日汇总）
           /   过去有用的资料   \
          /────────────────────\
         /       第4层          \  ← 深度搜索（按需触发）
        /  公司/部门/内幕信息    \    通过 MCP Server 实现
       /────────────────────────\
      /         第5层            \ ← 产业趋势（仅存档）
     /    行业大方向，背景参考     \
    /──────────────────────────────\
```

| **免费资源** | 免费云服务器、学生云服务器、阿里云学生、腾讯云学生、华为云学生、GitHub学生包、开发板免费/申请、硬件免费申请 | 领服务器、领硬件、领开发资源 |

## 第四层：深度搜索

**对应需求**：立刻就能获取信息的能力（公司实际情况、部门关系、少人知信息）

**实现方式**：通过 MCP Server 按需搜索（不放在频率词中自动匹配）

### 常用 MCP 工具组合

| 场景 | 工具调用 | 说明 |
|------|----------|------|
| 搜索公司信息 | `search_news(query="公司名", search_mode="entity", include_rss=True)` | 公司所有相关新闻 |
| 查找关联新闻 | `find_related_news(reference_title="公司名 事件", date_range="last_week")` | 找出相关报道 |
| 分析话题趋势 | `analyze_topic_trend(topic="公司名", analysis_type="trend")` | 热度变化趋势 |
| 情感分析 | `analyze_sentiment(topic="公司名")` | 正面/负面舆论 |
| 跨平台聚合 | `aggregate_news(date_range="today")` | 去重合并同一事件 |

### 使用场景示例

```
# 查华为最近的组织架构调整
search_news(query="华为", search_mode="entity", include_rss=True)

# 特斯拉最近有什么内部消息
find_related_news(reference_title="特斯拉 裁员", date_range="last_week")

# 小米和比亚迪的业务关系
search_news(query="小米 比亚迪", search_mode="fuzzy")

# 分析某个公司最近一个月的舆论趋势
analyze_topic_trend(topic="字节跳动", analysis_type="trend", date_range={"start": "2026-04-26", "end": "2026-05-26"})
```

| 层级 | 通知方式 | 触发条件 |

| 第一层（立即获利） | **实时推送** | 匹配即推送 |

## 关键词语法速查

| 语法 | 示例 | 说明 |
|------|------|------|
| 普通关键词 | `华为` | 标题包含即匹配 |
| 正则表达式 | `/\bAI\b/i` | 精确匹配英文单词 |
| 组别名 | `[华为]` | 给词组指定显示名称 |
| 必须词 | `+苹果` | 所有必须词都要匹配 |
| 排除词 | `!水果` | 匹配则排除 |
| 限制条数 | `@5` | 最多显示5条 |
| 别名 | `DeepSeek => DeepSeek` | 给关键词指定显示别名 |

## 优化策略

### 1. 关键词精准化
- 使用 `AI芯片` 而非 `芯片`
- 使用 `特斯拉 机器人` 而非 `特斯拉`
- 利用正则表达式精确匹配：`/\bAI\b/i` 避免匹配 MAIL

### 2. 来源筛选建议

**优先使用的 RSS 源**：
- Phoronix、LWN.net、Hackaday → Linux/嵌入式/开源硬件新闻
- TechCrunch AI、MIT Tech Review → AI 产业动态
- 机器之心、量子位 → AI 领域深度报道

**建议降级或禁用的源**：
- 今日头条、抖音热搜（噪声极大）
- 微博热搜（娱乐化严重）
- 百度热搜（广告多）

### 3. AI 二次过滤
- 设置相关性阈值 ≥ 6 才进入知识库
- 自动标记标题党
- 人工反馈闭环：钉钉通知里有点赞/忽略按钮

### 4. 定期回顾与微调
- **每周 15 分钟**：浏览推送信号，标记重要/误报
- **每月一次**：检查新出现的重要技术名词或公司

## 配置文件说明

| 文件 | 路径 | 用途 |
|------|------|------|
| 关键词配置 | `search_information/TrendRadar/config/frequency_words.txt` | 系统运行的关键词配置 |
| 主配置 | `search_information/TrendRadar/config/config.yaml` | 热榜平台、RSS源、通知、AI分析 |

**修改方法**：
1. 编辑 `frequency_words.txt` 文件
2. 重启 TrendRadar 服务（本地或服务器）
3. 系统会自动加载新的关键词配置
