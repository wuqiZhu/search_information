# 嵌入式Linux技术趋势监控系统

针对嵌入式Linux开发、BSP工程师、Linux驱动开发的技术趋势监控和处理系统。

最后更新：2026-05-21

---

## 项目概览

本系统由两个项目组成：

| 项目 | 路径 | 功能 |
|------|------|------|
| **search_information** | `search_information/` | 信号采集（TrendRadar + n8n） |
| **analyse_information** | `analyse_information/` | 数据处理（RSS拉取 + AI分析 + 知识库） |

---

## 项目一：search_information（信号采集）

### 架构概览

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  TrendRadar │───>│    n8n      │───>│  通知推送   │
```

### 启动命令

#### 一键启动
```bash
cd search_information
start.bat
```

#### 手动启动
```bash
# 安装依赖
cd trendradar
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 启动 n8n（Docker）
cd ../n8n
docker-compose up -d

# 启动 TrendRadar
cd ../TrendRadar
python main.py
```

### 通知配置

#### 钉钉通知
- 配置文件：`TrendRadar/config.yaml` 中 `notifications.dingtalk`
- 环境变量：`.env` 中 `DINGTALK_WEBHOOK` 和 `DINGTALK_SECRET`
- 钉钉机器人安全模式：**关键词**（关键词为 `通知`）
- 消息模板中已包含"通知"关键词

#### 邮箱通知
- 配置文件：`TrendRadar/config.yaml` 中 `notifications.email`
- 环境变量：`.env` 中邮箱相关配置

#### Webhook
- URL：`http://localhost:5678/webhook/trendradar`（推送到 n8n）

### 运行机制

| 功能 | 说明 |
|------|------|
| 信号抓取 | 每 30 分钟一轮 |
| 合并推送 | 10 分钟窗口内合并同类信号 |
| 每日报告 | 每天 20:00 发送 |
| 健康检查 | 每 24 小时检查信号源状态 |
| 数据清理 | 7 天后自动清理 |

### 已知问题与修复记录

| 问题 | 原因 | 修复方案 |
|------|------|----------|
| n8n 无法访问 | Docker容器未映射端口 | 用 docker-compose 重启 |
| 获取 0 信号 | 关键词太长、阈值太高 | 改为短词匹配，阈值降至 0.15 |
| 程序卡住不动 | RSS源无超时 | 添加 socket.setdefaulttimeout(30) |
| 首轮不发通知 | last_merge_time 初始化问题 | 初始化为 1 小时前 |
| 钉钉发送失败 | 消息缺少"通知"关键词 | 所有消息标题添加"通知" |
| V2EX 源极慢 | V2EX 响应超过 40 秒 | 已禁用 V2EX 源 |
| RSSHub 不可用 | rsshub.app 无法访问 | 移除 RSSHub 依赖，使用直接 RSS 源 |

## 常见问题

### Q: Docker镜像拉取失败怎么办？
A: 使用国内镜像源，编辑Docker配置文件，添加镜像加速器。

### Q: API调用失败怎么办？
A: 检查 API Key 是否正确，网络是否可访问 API 服务。

### Q: 如何修改监控关键词？
A: 编辑 `TrendRadar/config.yaml` 中的 `keywords` 部分。

### Q: 如何添加新的信号源？
A: 编辑 `TrendRadar/config.yaml` 中的 `sources` 部分。

### Q: 如何查看处理日志？
A: 查看 `TrendRadar/logs/trendradar.log` 文件。

### Q: 钉钉通知收不到？
A: 检查钉钉机器人安全模式，确保消息中包含机器人要求的关键词（当前为"通知"）。
