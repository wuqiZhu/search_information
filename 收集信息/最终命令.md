### 1. 检查服务是否真的在跑（尤其是重启后）

// 重要：`Win + R`，输入 `taskschd.msc`，回车

- **任务计划程序**不会自动弹窗提醒你服务失败，建议每周至少看一次：

  powershell

  ```
  # 查看 Docker 容器（n8n）
  docker ps
  
  # 查看 Python 进程（TrendRadar）
  Get-Process python -ErrorAction SilentlyContinue
  ```

  

- 如果进程没了，可以手动重启（参考 `精简命令.md` 的“启动服务”部分）。

------

### 2. 查看钉钉 & Obsidian 确认产出

- **钉钉群**：确认每天都能收到 P0/P1 信号和每日报告。如果连续半天没消息，说明可能抓取或推送异常。
- **Obsidian 知识库**：打开 `knowledge_base/obsidian/`，确认新文章按分类归档了。如果连续几天没有新文章，检查 RSS 源或 AI 分析是否正常。

------

### 3. 关注日志（尤其是刚配置完的头几天）

- 每周扫一眼日志末尾，看是否有报错：

  powershell

  ```
  # TrendRadar 日志
  Get-Content C:\...\TrendRadar\logs\trendradar.log -Tail 30
  
  # analyse_information 日志
  Get-Content C:\...\analyse_information\shared\logs\scheduler.log -Tail 30
  ```

  

- 常见可忽略警告：网络超时、个别 RSS 源解析失败。

- **需处理的错误**：API Key 失效、钉钉 Webhook 被禁、Docker 容器退出。

------

### 4. 修改配置后的必要手动操作

- **修改关键词 / 信号源**（`TrendRadar/config.yaml`）后，必须**重启 TrendRadar** 才能生效：

  powershell

  ```
  taskkill /F /IM python.exe
  cd C:\...\TrendRadar
  python main.py
  ```

  

- **修改 analyse_information 配置**（`analyzer/config.yaml`）后，不需要重启（每次运行重新读取），但可以手动跑一次 `--rss` 验证。

------

### 5. 定期备份（建议每月一次）

自动任务不会帮你备份，建议手动执行文档里的备份命令，或写一个简单的 PowerShell 脚本每月跑一次。
**最少要备份**：`.env` 文件、`TrendRadar/config.yaml`、`analyse_information/analyzer/config.yaml`、`knowledge_base/` 目录。

------

### 6. 手动触发场景（偶尔需要）

- 想立刻抓取最新信号：手动运行 `python main.py`（会自动跑一轮，不影响定时任务）。
- 想立刻分析某篇文章：`python analyzer/pipeline.py --url "https://..."`
- 想测试钉钉是否正常：用文档里的“测试钉钉通知”命令。

------

### 总结：自动化后你只需要做三件事

1. **看钉钉 + Obsidian**（确认有数据）
2. **每周瞄一眼日志**（确认没报错）
3. **改配置后重启 TrendRadar**（其他基本不用管）
